# Mini扇風機システム — 配線図・ブレッドボード配置ガイド

> このドキュメントは、`mini_fan_system` のコードをそのまま動かすための
> **物理的な配線図・ブレッドボード配置** を、初心者でも迷わないレベルで詳細化したものです。
>
> 配線対象：Arduino UNO R3 ＋ ELEGOO キット付属パーツ ＋ ブレッドボード（400 タイ以上）

> 📌 **ビジュアル配線図（先にこちらを見ることを推奨）**：[wiring_diagram.svg](wiring_diagram.svg)
> ブラウザ（または VSCode の Preview）で開くと、全部品・全配線・全孔位置が色付きで一目で分かります。本 markdown は補足説明用。

> ⚠ **2026-05-27 修正**：ELEGOO キット付属の 10kΩ ポテンショメータは **3296W 型**で、3 ピンが **三角配置**です。
> 一列には挿せません。正しい孔位置は **15-a（ワイパー）／ 14-e ／ 16-e**。詳細は §6-4 を参照。

### 電源レールの接続（重要）
- **Arduino 5V** を **左 + レール** に赤線で接続
- **Arduino GND** を **左 − レール** に黒線で接続
- 信頼性を上げたい場合：**左 + レール と 右 + レール** を赤線でブリッジ、**左 − レール と 右 − レール** を黒線でブリッジ
  （こうすると、ブレッドボードのどの位置からでも 5V/GND を取り出せます）

> **凡例**
> - `●` ： 部品の足やジャンパー線が刺さる穴（タイポイント）
> - `━━━` ： ジャンパー線・部品の本体
> - `[ ]` ： 抵抗・ダイオード本体
> - `‖` ： ブレッドボード中央の溝（左右の列を電気的に分離）
> - `│` ： 電源レール（縦方向に連続）

## 6. 部品ごとの詳細配線

### 6-1. 赤 LED（停止・異常表示）

**ブレッドボード上で使う具体的な穴位置**

```
    a    b    c    d    e   ‖   f    g    h    i    j     右側 GND レール
   ────────────────────────────────────────────────       ─────────
2  ●━━━━[220Ω 抵抗]━━━━●   ‖  ●━LED━●                       ●
   ↑                    ↑   ‖  ↑     │                       ↑
   D6 黄線              (連)│  長足  │                       │
                            ‖  (＋)  │                       │
3                            ‖  ●←(短足)●━━━黒ジャンパ━━━━●
                                 (－)
```

**穴位ごとの接続表（赤 LED 1 個に必要な部品・配線は 4 点）**

| # | 始点 | 終点 | 何で繋ぐ |
|:--|:--|:--|:--|
| ① | Arduino D6 | 行 2-a | **黄色ジャンパ線** |
| ② | 行 2-a | 行 2-e | **220Ω 抵抗**（a-e にまたぐ）|
| ③ | 行 2-f | 行 3-f | **赤 LED 本体**（長足＝アノードを 2-f に、短足＝カソードを 3-f に）|
| ④ | 行 3-j | 右側 GND レール（行 3 と同じ高さの穴） | **黒色短ジャンパ線** |

> 行 3 の f, g, h, i, j の 5 穴はブレッドボード内部で連通しているため、**短足を 3-f に挿し、黒線を 3-j から GND レールに引く**ことでカソードを GND に落とせます。

**なぜジャンパ ④ が必要？**
GND レールは**右側の長辺**にあり、行 3 の f〜j（端子区）と**直接は繋がっていない**ため、ジャンパ線で「橋渡し」する必要があります。

**ポイント**
- LED は極性あり：**足の長い方が +（アノード）／短い方が －（カソード、平面側）**、逆だと点灯しない
- 抵抗 220Ω は LED と直列なら、抵抗の向きはどちらでも OK
- 行内では a-e（左半区）／f-j（右半区）がそれぞれ連通、中央の溝 `‖` が左右を分離

### 6-3. タクトスイッチ（ON/OFF ボタン）

```
   Arduino D2 ──黄線──→ ● 行10-a
                          │
                          ●─ボタン左上ピン
                          
                  ┌───────────────┐
                  │ タクトスイッチ │
                  │   ┌─┐         │
                  │   │ │         │  ← この本体（4 ピン）
                  │   └─┘         │
                  └───────────────┘
                          │
                          ●─ボタン右下ピン
                          │
                          ●─→ 行13-a
                          │
                       黒線
                          │
                          ▼
                       GND レール（青）
```

**ポイント**
- タクトスイッチは 4 ピンだが、**対角線上の 2 ピンだけ使えば OK**（上下/左右に同じピンが繋がっている）
- ブレッドボードの溝（左右分離）をまたぐように刺すと、左 2 ピンと右 2 ピンが分離される正しい挿し方になる
- INPUT_PULLUP を使うので **外付け抵抗は不要**（Arduino 内蔵プルアップを利用）

### 6-4. ポテンショメータ（風量つまみ）⚠ 三角配置

ELEGOO キット付属の 10kΩ ポテンショメータは **3296W 型（青色トリマー）** で、
3 ピンが**一直線ではなく三角形に配置**されています。1 本だけ離れたピンが片側、
隣接した 2 本が反対側にあります。

```
   実物上面図（上部に十字ネジ）：

           ┌─────────────────┐
           │ 10kΩ トリマー   │
           │     ╋ ネジ      │
           │                 │
           └─┬─────────┬──┬──┘
             │         │  │
             ●         ●  ●
          (wiper)    (端1)(端2)
          単独1本      隣接2本
```

**ブレッドボード挿入位置（正しい三角配置）**

| ピン | 挿入孔 | 配線 |
|:--|:--|:--|
| **wiper**（単独1本） | **15-a** | 🔵 青線 → Arduino A0 |
| **端 1**（隣接2本の片方） | **14-e** | 🔴 赤線 → 左側 + レール |
| **端 2**（隣接2本のもう片方） | **16-e** | ⚫ 黒線 → 左側 − レール |

**ポイント**
- ❌ **誤り**：3 ピンを 15-a / 15-c / 15-e に並列で挿す（物理的に不可能。ピンが曲がる）
- ✅ **正解**：三角分布の孔 **14-e / 15-a / 16-e**（行差 2、列差 4 で離れた位置）
- ワイパーが A0 へ。ネジを回すと 0V〜5V が連続的に出る
- 端 1 / 端 2 のどちらを 5V／GND にしても動作する。シリアルで `pot:` を見て
  - 時計回り → 数値増 = OK
  - 時計回り → 数値減 = 14-e と 16-e の 2 本を入れ替える

### 6-5. PN2222 NPN トランジスタ（モーター駆動）

```
   PN2222 を「平らな面を自分側に向けて」見たとき：
   
              ┌─────┐
              │ FLAT│  ← フラット面（型番が書いてある）
              │     │
              └─────┘
              E  B  C
              │  │  │  ← 左から エミッタ（E）、ベース（B）、コレクタ（C）
              ●  ●  ●
              
   配線：
   E（エミッタ）── 黒線 ──→ GND レール（青）
   B（ベース）  ── ●─[1kΩ]─●── 緑線 ──→ Arduino D9（PWM）
   C（コレクタ）── 白線 ──→ モーターの (-) 端子
```

**ポイント（最重要）**
- ピン配置：「**Eベース左、ベース真ん中、コレクタ右**」 ※平らな面を手前に向けて
- 逆刺しすると壊れる／動かないので、フラット面の向きに必ず注意
- ベースには **必ず 1kΩ 抵抗** を入れる（直結すると過電流で焼ける）

### 6-6. DC モーター ＋ Diode Rectifier（逆起電力保護）

> このセクションが最もミスしやすい場所です。**ダイオードをモーターの両端に「またがせる」**ように刺し、**白帯を +5V 側に向ける**のがポイントです。

**目指す回路の概念図**

```
                   +5V レール ──┬──────────────●  モーター (+)
                                │                ┌─────────┐
                           [ダイオード]          │  MOTOR  │
                           │ 白帯                │ Fan     │ ← 物理モーター（ブレッドボード外）
                           │ (カソード)         │  Blade  │
                           │                    └─────────┘
                           │ アノード               │
                                │                    │
                       PN2222   ●──────────────●  モーター (-)
                       コレクタ
```

**ブレッドボード上での具体的な実装**

**2 行（行 24 と行 25）を「モーター両端の引き出し板」として使います**：

```
        a   b   c   d   e   ‖   f   g   h   i   j        右側 + レール
       ─────────────────────────────────────────         ─────────
                  ↓ (行 20-c の PN2222 コレクタから)
24 ●━━━━━━━━━━━━━━━●━━━●  ‖  ●━━━●━━━●━━━●━━━●               （この行＝「モーター (-) 側」）
                            ‖  ↑      ↑
                            ‖  ダイオ 引き出し線→モーター (-)
                            ‖  ードの
                            ‖  アノード
                            ‖  ↑
                            ‖  │
                            ‖  ↓ ←── ダイオードを縦に 24-25 行にまたぐ向きで挿入
                            ‖  白帯
                            ‖  (カソード)
25                          ‖  ●━━━●━━━●━━━●━━━●━━━━━━●     （この行＝「モーター (+) 側」）
                            ‖              ↑              ↑
                            ‖              引き出し線→     + レールへ
                            ‖              モーター (+)
```

**詳細配線手順（5 本のジャンパー＋部品）**

| # | 始点 | 終点 | 何で繋ぐ |
|:--|:--|:--|:--|
| ① | 行 20-c（PN2222 コレクタ）| 行 24-f | **白色ジャンパ線**（コレクタを行 24 まで引く）|
| ② | 行 24-g | 行 25-g | **ダイオードを縦に挿入**、**アノード**を 24-g、**白帯（カソード）**を 25-g |
| ③ | 行 24-j | **モーター (-) 端子** | **白色長ジャンパ線**（公-メス、または鰐口クリップ）|
| ④ | 行 25-j | **モーター (+) 端子** | **赤色長ジャンパ線** |
| ⑤ | 行 25-e（または d）| 右側 + レール | **赤色短ジャンパ線**（モーター給電）|

**配線後の電流の流れ**

| 状態 | 電流の経路 | ダイオードの状態 |
|:--|:--|:--|
| モーター ON（PN2222 導通）| +5V レール → 行 25 → モーター (+) → モーター (-) → 行 24 → コレクタ → エミッタ → GND | 逆バイアス、**非導通** |
| モーター OFF（PN2222 遮断）| 電流なし | 作用なし |
| OFF 切替の瞬間 | コイルが逆向き電圧を生成（-側が瞬間的に +側より高くなる）→ 行 24 → アノード → 白帯 → 行 25 → +5V レール へ吸収 | **順バイアス、導通**（PN2222 を保護）|

**なぜダイオードはこの向き？（白帯を + 側へ）**
- 通常運転時：+5V が行 25（白帯側）、コレクタはほぼ 0V で行 24（アノード側）
- ダイオードは「カソード側が高電位／アノード側が低電位」のとき**非導通**で、PWM 動作の邪魔をしない
- モーター遮断の瞬間、インダクタが逆起電圧を発生（行 24 が瞬間的に高電位、行 25 が低電位）→ ダイオードが**導通**して逆方向パルスを安全に +5V へ逃がす
- **逆向きに付けると**：通常運転時に +5V と GND が短絡し、ダイオードと PN2222 が瞬時に焼損する

**ポイント（最重要）**
- **ダイオードの 2 本の足を行 24 と行 25 の同じ列（g 列）に挿し**、ダイオードが 2 行をまたぐようにする
- **白帯は必ず行 25（+5V 側）**、アノードは行 24（コレクタ側）に
- モーターの 2 本のリード線は行 24-j（- 端）と行 25-j（+ 端）から外部へ引き出す。**モーター本体はブレッドボードに乗せず、外部に置く**
- 共通 GND の保証：Arduino GND ⇄ ブレッドボード GND レール ⇄ PN2222 エミッタ（行 20-a）

## 7. 配線手順（推奨順）

ステップごとに小さく動作確認しながら進めると、原因特定が楽です。

### ステップ 1：電源レール接続（5V/GND）
1. Arduino の **5V** → ブレッドボード上側の **+ レール**（赤線）
2. Arduino の **GND** → ブレッドボード上側の **- レール**（黒線）
3. 念のため、上 - レールと下 - レールを黒線でブリッジ（共通 GND）

### ステップ 2：赤 LED 配線（D6）
1. D6 → 行 2-a へ黄線
2. 行 2 の a-e 間に 220Ω 抵抗
3. 行 2-f に LED アノード（長い足）、行 3-f に LED カソード
4. 行 3-f から - レールへ黒線

→ **動作確認**：Arduino に電源を入れる（コード書き込み済みなら）と、起動直後に赤 LED が点灯するはず

### ステップ 3：緑 LED 配線（D5）
赤 LED と同じパターンで、D5 → 220Ω → 緑 LED → GND

→ **動作確認**：ボタン未配線でも、`updateLED()` のテストで点灯確認可能

### ステップ 4：タクトスイッチ配線（D2）
1. ボタンを行 10-11 にまたがるよう挿す（**中央溝をまたぐ向き**）
2. ボタン左上ピン（行 10-a）→ D2 へ黄線
3. ボタン右下ピン（行 11-j）→ - レールへ黒線

→ **動作確認**：シリアルモニタで状態変化を確認、押すたび OFF↔運転中 が切り替わる

### ステップ 5：ポテンショメータ配線（A0）— ⚠ 三角配置注意

ELEGOO の 3296W 型ポテンショメータは 3 ピンが三角形に並んでいるので、**同一行には挿せません**。

1. **単独1本のピン（ワイパー）** を **15-a** に挿す
2. **隣接した2本のピン** を **14-e** と **16-e** に挿す
3. **15-a**（wiper） → A0（青線）
4. **14-e**（端 1） → + レール（赤線）
5. **16-e**（端 2） → − レール（黒線）

→ **動作確認**：シリアルモニタで `pot:` の値を見ながらつまみを回すと、0〜1023 で変化する。
   方向が逆（時計回りで減る）なら、14-e と 16-e の 2 本を入れ替える。

### ステップ 6：PN2222 トランジスタ配線（D9）
1. PN2222 を行 20-21 に挿す（**平らな面を自分側に**、E が左 / 20-a）
2. エミッタ（20-a）→ - レールへ黒線
3. ベース（20-b）→ 行 20-c, d, e のどこかに 1kΩ 抵抗
4. その抵抗の反対端（例: 20-e）→ D9（緑線）

### ステップ 7：モーター + ダイオード配線
1. PN2222 コレクタ（20-c）から白線でブレッドボード適当な行（例: 24-f）へ
2. その行（24-f, g）にダイオードのアノード（白帯がない側）を挿す
3. ダイオードのカソード（白帯側）を別の行（例: 25-f, g）へ
4. モーター (-) → 24-h（コレクタ側と同じ行）
5. モーター (+) → 25-h（白帯側と同じ行）
6. 25 行から + レールへ赤線（モーター駆動電源）

→ **動作確認**：ボタンを押して運転中にし、つまみで回転速度が変わることを確認

## 9. トラブルシューティング

| 症状 | 原因候補 | 対処 |
|:--|:--|:--|
| 何も動かない・LED 点かない | USB 給電されていない／GND 未接続 | USB ケーブル・GND 配線確認 |
| 赤 LED は点くが緑 LED が点かない | D5 配線、緑 LED 極性逆 | ジャンパー確認、LED 向き反転 |
| ボタンを押すと毎回 2 回切り替わる | デバウンス未動作（コード問題） | コード側 `DEBOUNCE_MS` 確認 |
| 起動直後にいきなり運転中になる | ボタンが押された状態で起動 / 配線で短絡 | `buttonReady` 動作確認・配線見直し |
| つまみを回しても速度が変わらない | A0 配線、中央ピン誤り | ポテンショ中央が A0 か確認 |
| モーターが「ジー」と唸るが回らない | PWM 値が `MOTOR_MIN_PWM` 未満 | コード設計通り（つまみを上げる） |
| モーター OFF にした瞬間に Arduino がリセットする | ダイオード未配線・逆向き | ダイオード方向確認（白帯を +5V 側） |
| シリアルモニタに何も出ない | ボーレート不一致 | シリアルモニタを 9600 bps に設定 |
| 一瞬モーターが回るがすぐ止まる | 電流不足（USB 給電の限界） | 外部 5V 電源を検討（モーターのみ） |

## 10. 安全・注意事項

- **モーターの電源を Arduino 5V から取る場合**、USB 給電の電流容量（約 500mA）に注意。
  小型ファンモーターでも起動瞬間に大電流が流れることがあり、
  Arduino がリセット・USB が切断される場合は **外部 5V 電源** を使う（共通 GND は必須）。
- **配線中は USB を抜く**。電源を入れたまま配線変更すると、ショートで部品が焼ける可能性あり。
- **ダイオードの向きを必ず確認**。逆向きに付けた状態で電源を入れると、ダイオード自身も
  PN2222 も焼損するリスクがある。
- **PN2222 の足が熱くなった**ら、すぐ電源を切る（過電流の兆候）。
