## 1. システム全体像

### 1-1. ブロック構成（入力 → 処理 → 出力）

> ※ Arduinoを中心に「何が入力で何が出力か」をテキストの図で整理します。

**記入例：**
```
【入力】                 【Arduino UNO R3】            【出力】
タクトスイッチ ─────→┐                         ┌──→ LED（赤・緑）
超音波センサー ─────→┤   状態管理・判定処理    ├──→ パッシブブザー
                      └─────────────────────────┘
```

**↓ 自分のシステムを書いてください**
```
【入力】                 【Arduino UNO R3】            【出力】
DHT11温度センサー(D2) ─→┐                         ┌──→ モーター(D9 PWM/停止)
IR受信モジュール(D11) ─→┤  モード管理・温度判定   ├──→ LED赤(D3): 停止/電源OFF通知
音/光センサー(A0 任意) ─→┤  (自動/手動/電源ON-OFF) ├──→ LED緑(D4): 通常回転/電源ON通知
                        ┤  (温度しきい値, PWM制御)├──→ LED黄(D5): 強回転
                        └─────────────────────────┘
```

補足説明（図で表現しきれない場合）：
- 自動モードでは温度しきい値に応じてD9のPWMデューティを切り替える（例: 25℃未満=弱、25〜30℃=中、30℃以上=強）。
- 手動モードではリモコン入力を優先し、温度制御を一時停止して風量を固定する（0=弱、1=中、3=強）。
- 電源キー入力時は電源状態を切り替え、OFF中はモーター出力を停止する。
- 任意機能として、音/光センサーが設定しきい値を超えたときに電源ON/OFF制御を行う（実装は余裕がある場合のみ）。
- モーターはArduinoに直結せず、D9のPWM信号をトランジスタ駆動回路へ入力して制御する。
- 電源ON/OFF時はLEDを短く点滅させて状態通知する（ON時: 緑を2回点滅、OFF時: 赤を2回点滅）。

### 1-2. 状態遷移（ガジェットの「ふるまい」設計）

> ※ ガジェットが「どんな状態を持つか」を定義します。
> この設計が loop() の骨格になります。

**記入例：**
```
[電源ON / 初期化]
  ↓（初期化完了）
[待機中] ──（センサー検知）──→ [計測中] ──（閾値超え）──→ [警告中]
   ↑                                                            │
   └────────────────────（リセットボタン）────────────────────────┘
```

**↓ 自分の状態遷移を書いてください**
```
[電源ON / 初期化]
  ↓（DHT11/IR初期化OK）
[自動制御]
  │  ├─（温度<25℃）────────────→ [弱風]
  │  ├─（25〜30℃）────────────→ [中風]
  │  └─（30℃以上）───────────→ [強風]
  │
  ├─（IR: MODEキー）─────────→ [手動制御]
  ├─（IR: 電源キー）─────────→ [電源OFF]
  └─（DHT11読取失敗2回連続）──→ [異常保護]

[手動制御]
  ├─（IR: 0/1/3キー）────────→ [弱風/中風/強風に固定]
  ├─（IR: MODEキー）─────────→ [自動制御]
  ├─（IR: 電源キー）─────────→ [電源OFF]
  └─（DHT11読取失敗2回連続）──→ [異常保護]

[任意: センサー電源制御]
  ├─（音/光センサー値がしきい値超過）──→ [電源OFF]
  └─（音/光センサー値が復帰しきい値を下回る）→ [自動制御]

[電源OFF]
  ├─（IR: 電源キー）─────────→ [自動制御]
  └─（その他キー）──────────→ [電源OFFを維持]

[異常保護]
  ├─（isPowerOn=true）───────→ [中速固定]
  ├─（isPowerOn=false）──────→ [停止固定]
  ├─（正常値3回連続 かつ isPowerOn=true）──→ [自動制御]
  └─（正常値3回連続 かつ isPowerOn=false）─→ [電源OFF]
```

モード切替キー定義（実装前に固定する）:
- MODEキー: 自動制御と手動制御を相互に切替
- 電源キー: ON/OFF切替（OFF中はモーター停止）
- 0/1/3キー: 手動制御中のみ弱/中/強を指定

**各状態の意味：**

| 状態名 | この状態のとき、何をしているか | LEDやブザーなど出力の様子 |
|:--|:--|:--|
| 初期化 | ピン設定、IR受信開始、DHT11初回読取確認 | 赤/緑/黄を100msずつ順番点灯（起動確認） |
| 自動制御 | 1秒ごとに温度取得し、しきい値で風量を弱/中/強へ更新 | 弱/中=緑点灯、強=黄点灯 |
| 手動制御 | リモコンキーで風量固定（0=弱, 1=中, 3=強）し温度制御を停止 | 風量に対応してLED点灯、MODEキーで自動へ戻る |
| 電源OFF | 出力停止状態を維持し、電源キー再入力のみ受け付ける | 赤点灯（モーター停止） |
| 異常保護 | DHT11異常時に安全側固定（電源ON時: 中速固定、電源OFF時: 停止固定）。正常値3回連続でisPowerOnに応じて自動制御/電源OFFへ復帰 | 赤LEDを2Hzで点滅（異常通知） |

> [!TIP]
> すべての状態から「抜け出せるルート」があるか確認してください。
> 「このボタンを押せば必ずリセットに戻る」が保証されていますか？

### 2-3. タイミング設計（delay vs millis）

> ※ `delay(N)` はN ms間、**すべての処理**が止まります。
> 複数の処理を並行させたい場合は `millis()` を使います。

| 処理 | 必要な周期 | delay / millis | 判断の理由 |
|:--|:--|:--|:--|
| 温度センサー読み取り（DHT11） | 1000ms | **millis** | DHT11の仕様上、過剰ポーリングを避ける |
| IR受信監視 | 10ms相当（毎loop） | **millis** | リモコン操作の取りこぼし防止 |
| モーターPWM反映 | 状態変更時即時 | **millis** | 遅延なく風量を更新する |
| 異常時LED点滅 | 500ms | **millis** | 異常通知しつつ他処理を停止しない |

> [!CAUTION]
> `delay()` を使う場合、その間はボタン入力もセンサー読み取りも**すべて止まります**。

### 3-1-2. 詳細回路構成図（ブレッドボード配置 + PIN番号）

```
【Arduino UNO R3】
D2  ───────────────→ DHT11 DATA
D11 ───────────────→ IR Receiver OUT
A0  ───────────────→ 音/光センサー OUT（任意）
D3  ──220Ω──→ LED赤(+)   LED赤(-) ─→ GND
D4  ──220Ω──→ LED緑(+)   LED緑(-) ─→ GND
D5  ──220Ω──→ LED黄(+)   LED黄(-) ─→ GND
D9  ──1kΩ───→ Tr Base
GND ───────────────→ Tr Emitter
5V  ───────────────→ モーター(+) / DHT11 VCC / IR VCC
Tr Collector ─────→ モーター(-)
モーター両端にダイオード(1N4007): カソード=5V側, アノード=Collector側

【ブレッドボード配置（上面イメージ）】
上側+レール: 5V共通ライン（DHT11 VCC, IR VCC, モーター+）
上側-レール: GND共通ライン（DHT11 GND, IR GND, LED各カソード, Tr Emitter）

中央エリア左: DHT11モジュール（DATA列をD2ジャンパへ）
中央エリア中央: IR受信モジュール（OUT列をD11ジャンパへ）
中央エリア右: NPNトランジスタ（EをGND列、Cをモーター-列、Bを1kΩ経由D9へ）
右端列: 赤/緑/黄LEDを縦配置（各アノードを220Ω経由でD3/D4/D5へ）
```

| ブレッドボード位置 | 配置部品 | 接続先 |
|:--|:--|:--|
| 上側電源レール（+） | 5Vライン | UNO 5V, DHT11 VCC, IR VCC, Motor+ |
| 上側電源レール（-） | GNDライン | UNO GND, DHT11 GND, IR GND, LED-, Tr E |
| 中央左列 | DHT11 | DATA→D2 |

## 7. AIレビュー記録

> グループレビューの前に必ず実施してください。

### Q1: 設計の妥当性確認

> 「この基本設計書をもとに Arduino UNO R3 でコードを書きます。ピン競合・ライブラリ競合・状態遷移の抜け・delay() の問題点はありますか？」

**AIの回答（要約）：**
- ピン競合はない（D0/D1未使用、PWMはD9のみ）。
- `IRremote` と `DHT` の併用は可能だが、`delay()`多用時にIR取りこぼしが発生しやすい。
- 状態遷移は「異常保護→復帰条件」が未定義だとループ復帰不能になる。
- モーター制御はトランジスタ+フライバックダイオード必須。

**対応した内容：**
- 全周期処理を`millis()`ベースに統一し、`delay()`を設計上不使用とした。
- 状態遷移へ「異常保護」「正常値3回連続で復帰」を追加した。
- 3-1-2にトランジスタ駆動回路とダイオード接続を明記した。

---

### Q2: チャタリング・delay・ノイズの確認

> 「この設計にチャタリング対策は含まれていますか？delay() を使っている箇所で問題はありますか？アナログセンサーを使う場合、ノイズ対策は必要ですか？」

**AIの回答（要約）：**
- IR入力にも実質的なチャタリング（重複受信）があるため、最小受付間隔を設けるべき。
- DHT11は更新周期を1秒以上にしないと失敗率が上がる。
- アナログ入力は本設計で未使用だが、モーター駆動ノイズがGNDに乗る可能性があるため配線分離とGND共通点の整理が有効。

**対応した内容：**
- IR受信に50msデバウンス（同一コード再受付制限）を追加した。
- DHT11読取周期を1000ms固定に設定した。
- ブレッドボード配置で「電源レール共通」「信号線を電源線から離す」方針を追記した。
